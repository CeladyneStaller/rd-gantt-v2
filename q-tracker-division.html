Broker token
The tracker talks only to the broker — it reads and writes through it, and the broker holds the JSONBin key and bin IDs server-side. Only the token is required; the URL is pre-filled with the deployed broker, so change it only if the service moves. Both values are stored only in this browser (localStorage), never in the file. For SSO‑gated Confluence embeds, append ?token=<token> to the iframe URL and viewers never enter anything (it's seeded then stripped from the URL). If the broker is unreachable the tracker can show its last cached view but can't load fresh data.
